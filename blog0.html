How I found my first Cross Site Port Attack

Hello everyone, this is a story of how I found Cross Site Port Attack on a bug bounty program.
This is a private program and doesn't allow disclosure so let us assume that our target is redacted.com. This is a wide scope target. So, after running my subdomain enumeration scripts I passed the subdomains into httpx to get live subdomains.
I had over 14k unique targets after probing. At this point, I was searching for keywords like internal, staging, stg, developer, admin, etc.
While testing these interesting domains manually, I also ran the nuclei technologies template on the live subdomains to see what technologies the target was using. Upon running this and parsing the output, I found out that one of the subdomains let us say 'vuln.redacted.com' was built in WordPress.
I knew there were CVEs for weblogic. I ran the CVE templates from nuclei on 'vuln.redacted.com'. As I was expecting nuclei detected the most common xmlrpc.php file was enabled on'vuln.redacted.com'.
